# 常见使用场景

如果你不确定 PsyGo 适不适合你的工作，可以先从最接近日常任务的那一类开始看。

这页不是能力上限清单，而是最常见、最容易落地的使用方式。对于需要接入知识库、业务系统或外部平台的场景，通常要先完成授权和接入，PsyGo 才能真正开始干活。

## 日常办公与内容工作

### 文档写作与内容整理

适合把零散信息整理成可提交、可流转、可存档的正式内容。

可支持的具体场景：

- 会议纪要整理与行动项提炼
- 周报、月报、季度总结撰写
- 项目方案、实施方案、汇报材料编写
- 制度文档、SOP、操作手册撰写
- 需求文档、产品文档整理
- 合同初稿、商务函件、通知公告起草
- 长文档润色、扩写、缩写、改写
- 多资料整合成一份结构化文档

可交付结果：Word 文档、制度文件、方案文档、纪要、总结报告等。

### 表格处理与数据分析

适合处理重复度高、字段多、需要汇总和对比的表格类工作。

可支持的具体场景：

- Excel 数据清洗、去重、合并、格式规范
- 销售数据、运营数据、财务数据汇总
- 自动生成日报、周报、月报
- 指标统计、趋势分析、异常波动识别
- 多表关联分析、字段匹配、台账管理
- 预算测算、成本分析、利润分析
- 问卷数据、调研数据统计分析

可交付结果：Excel 分析表、可视化图表、经营报表、分析结论。

### 调研与报告生成

适合需要先查资料、再形成结构化结论的任务。

可支持的具体场景：

- 行业研究报告
- 竞品分析报告
- 市场趋势分析
- 用户画像与需求分析
- 政策研究与资料汇编
- 技术路线调研
- 商业模式分析
- 项目可行性研究报告
- 采购比选与供应商调研

可交付结果：调研报告、竞品分析、行业洞察、结论摘要。

### PPT 制作与汇报表达

适合把已有材料转成适合汇报、演示和培训的页面内容。

可支持的具体场景：

- 工作汇报 PPT
- 项目方案汇报 PPT
- 领导汇报材料
- 培训课件制作
- 年度总结 / 经营分析汇报
- 根据 Word / Excel 自动生成 PPT 大纲与页面内容

可交付结果：PPT 初稿、演讲提纲、图文页内容、汇报框架。

## 产品、研发与技术工作

### 软件开发与技术实现

适合从需求梳理到编码交付的完整研发流程。

可支持的具体场景：

- 需求梳理与 PRD 撰写
- 原型设计与页面流程说明
- 前端页面开发
- 后端接口开发
- 数据库设计
- 管理后台搭建
- 自动化脚本编写
- API 集成与系统对接
- Bug 排查与代码重构
- 测试用例生成与开发文档补全

可交付结果：代码、接口文档、开发方案、测试脚本、技术文档。在具备权限和部署条件时，也可以直接上线。

### 技术研究与方案设计

适合在正式开发前，先做选型、评估和方案论证。

可支持的具体场景：

- 技术选型建议
- 架构设计方案
- 开源项目调研
- 性能优化建议
- 故障定位与问题排查
- AI / Agent / 大模型技术研究
- 数据平台建设方案
- 安全与权限设计方案
- 系统改造与升级建议

可交付结果：技术调研报告、架构图、选型建议、解决方案。

## 管理与职能支持

### 项目管理与协同推进

适合需要持续跟踪进度、风险和里程碑的项目型工作。

可支持的具体场景：

- 项目计划拆解
- 任务排期与甘特图整理
- 风险点识别
- 进度周报自动生成
- 里程碑提醒与复盘总结

适合对象：PM、产品经理、项目负责人、团队管理者。

### 财务与经营分析

适合基于业务和财务数据做汇总、拆解和对比分析。

可支持的具体场景：

- 报销单据整理
- 财务报表摘要
- 预算编制辅助
- 成本 / 费用分析
- 利润结构分析
- 经营数据周报月报
- 多维度对比分析

适合对象：财务、经营分析、管理层。

### 法务与合规支持

适合做材料整理、条款比对和法规信息梳理，但不应被宣传为替代专业法务判断。

可支持的具体场景：

- 条款比对与风险提示
- 合规材料整理
- 政策法规检索与摘要
- 审核意见汇总

适合对象：法务、行政、管理部门。

> 对外宣传建议写“辅助处理”，避免过度承诺“替代专业法务判断”。

### 行政与综合办公

适合处理跨部门协同、流程推进和资料归档类事务。

可支持的具体场景：

- 通知公告撰写
- 制度文件整理
- 采购需求汇总
- 资产台账管理
- 活动组织与流程安排
- 出差行程整理
- 文件归档与分类检索

适合对象：行政、综合管理、办公室。

## 企业级知识与经营支持

### 知识库建设与企业知识管理

这类场景通常需要先接入外部知识库平台，再让 PsyGo 按统一规则整理和输出。

可支持的具体场景：

- 文档自动分类归档
- FAQ 自动整理
- 历史案例沉淀
- SOP 知识库搭建
- 内部制度统一管理
- 跨部门知识检索
- 新员工培训知识包生成

适合对象：全公司，尤其适合中大型企业。

### 企业经营决策支持

这类场景通常需要接入各部门系统，先打通数据，再做综合分析和汇报。

可支持的具体场景：

- 多部门数据整合
- 经营指标看板总结
- 异常指标预警
- 业务问题归因分析
- 管理层决策材料生成
- 战略会议资料准备

适合对象：CEO、COO、部门负责人、经营分析团队。

## 不知道从哪一类开始？

先按你最常见的交付物来选：

- 经常交 Word、纪要、方案，先看“文档写作与内容整理”
- 经常交 Excel、报表、图表，先看“表格处理与数据分析”
- 经常交 PPT、汇报材料，先看“PPT 制作与汇报表达”
- 经常交代码、接口、系统方案，先看“软件开发与技术实现”
- 经常做跨部门推进，先看“项目管理与协同推进”

如果你已经知道怎么给 PsyGo 下任务，可以回到 [快速上手](./intro) 对照任务写法；如果还不确定某个场景怎么提需求，最稳妥的做法仍然是把目标、背景、交付要求和限制条件一起说明。
