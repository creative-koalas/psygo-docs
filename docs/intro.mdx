# PsyGo 快速上手

把 PsyGo 当成一个刚入职、能力很强的远程实习生；把 PsyGo App 当成和 TA 沟通的微信。这样理解，基本就不会用错。

## 先记住这 4 件事

1. **PsyGo 很能干，但不会读心。** 任务越清楚，结果越接近你的预期。
2. **PsyGo 不是聊天机器人。** 你下完任务后，不需要一直等在 App 里。
3. **PsyGo 有自己的电脑。** 它看不到你本地设备上的内容，除非你主动发给它或授权给它。
4. **PsyGo 的交付通常是成果，不是聊天记录。** 更常见的是文件、链接、仓库地址，或者直接可用的网站。

如果你已经知道自己想交给 PsyGo 什么工作，可以直接看 [常见使用场景](./use-cases)，按任务类型快速找到最接近的用法。

## 怎么给 PsyGo 下任务

最简单的公式是：

**目标 + 背景 + 交付要求 + 限制条件**

比如，这样说通常不够清楚：

> 帮我做个 OPC 平台

换成这样，PsyGo 更容易做好：

> 帮我做一个湖北省武汉市江汉区 OPC 平台的介绍网站。先查相关政策，保证信息准确，标清信息来源，最后给我网站链接和源码仓库。

如果你们团队有固定做法，也可以先教一次，后面重复使用：

> 记住：我们部门做行业调研时，要先查资料，尽量使用近两年的信息；交付用 PDF，参考来源要标清楚。

之后你只需要说：

> 给我出一份 AI 游戏行业调研报告，按我之前教你的方式做。

## 下完任务后，不用守着 App

PsyGo 更像在远程干活的人，不像聊天机器人。你把任务说清楚以后，可以直接关掉客户端，先去做别的事。

PsyGo 做完了，或者需要你补充信息时，会再来找你。

## 你可以发给 PsyGo 什么

只要和工作有关，基本都可以发：

- Word 文档
- Excel 表格
- PPT
- PDF
- 网页链接
- 压缩包
- Git 仓库地址
- Token 或其他授权信息

PsyGo 也可能回传给你这些内容，或者直接给你一个可访问的网站、在线文档、代码仓库。

## PsyGo 看不到你的本地电脑

PsyGo 不运行在你的电脑上，每个 PsyGo 都有自己的工作环境。

这意味着：

- 你没发给它的文件，它看不到
- 你没告诉它的信息，它不知道
- 你没授权的系统和仓库，它用不了

所以，凡是任务需要的资料、文件、账号权限，都要明确提供给它。

## 不要把 PsyGo 当成什么

PsyGo 不是豆包、ChatGPT 这类聊天机器人，也不是在你本地操作的开发工具。

不建议这样用：

- 让它做很复杂的工作，但要求它全程只在聊天框里汇报
- 让它直接操作你本地电脑、给你本地配环境
- 把一个完整任务拆成很多极小步骤，像用命令行工具一样一条条指挥

更有效的方式是：

- 直接给完整目标，再补充关键约束
- 让它自己决定执行细节
- 需要时再像带实习生一样补充指导

把 PsyGo 当成一个远程工作的全能实习生：给清楚目标，提供必要信息，然后让 TA 去干活。
